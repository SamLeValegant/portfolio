Gestion des sessions de jeux pour L'Approximatographe - Python CustomTkinter

Système centralisé de gestion des sessions de jeux coopératifs pour l'association L'Approximatographe. Ce projet a permis de centraliser la gestion en temps réel des sessions de jeux, sauvegarder les données statistiques et établir une communication bidirectionnelle entre l'interface de gestion CustomTkinter et les modules Godot distribués sur plusieurs ordinateurs.

Contexte du Projet

L'Approximatographe est une association angevine proposant 5 jeux coopératifs nomades, éthiques, durables et inclusifs. Initialement, les jeux Godot étaient des modules indépendants sur différents ordinateurs sans communication entre eux, ce qui impliquait :

  • Installation longue et gestion manuelle
  • Perte de données des sessions passées
  • Absence de centralisation et de sauvegarde
  • Impossibilité de gérer une session de façon automatique

Architecture Globale

Infrastructure : Réseau local filaire avec Raspberry Pi 5 comme nœud central hébergeant l'interface Python CustomTkinter, le serveur Python, Apache2 et MariaDB. Quatre postes clients Linux exécutent les jeux Godot, tous reliés via switch avec adresses IP statiques.

Communication : Protocole TCP garantissant la livraison ordonnée des messages. L'interface envoie des signaux de démarrage/arrêt aux modules Godot via le serveur Python, qui reçoit les variables des jeux et les écrit dans un fichier texte lu par l'interface.

Fonctionnalités principales

  • Gestion en temps réel des sessions avec score dynamique et chronomètre
  • Interface Python CustomTkinter responsive avec architecture MVC
  • Sauvegarde automatique des données des sessions en base de données
  • Communication TCP bidirectionnelle entre interface et modules Godot
  • Système de missions configurable par jeu
  • Affichage dynamique des variables reçues des postes clients
  • Partie statistiques avec graphiques et données compilées
  • Gestion des jeux, lieux et compléments via interface admin

Technologie utilisée

Raspberry Pi 5 Python 3 CustomTkinter MariaDB Apache2 phpMyAdmin TCP/IP Godot/GDScript

Détails Techniques

Base de données : 5 tables relationnelles (SESSIONS, JEUX, LIEUX, COMPLEMENTS, LIEUXCOMPLEMENTS) avec contraintes ON DELETE NULL pour l'intégrité référentielle.

Interface CustomTkinter : Architecture MVC en Python avec séparation claire entre Models (BDD), Controllers (logique métier) et Views (interface utilisateur). Widgets réutilisables pour cohérence visuelle moderne.

Serveur Python : Écoute sur port 5000 en TCP. Reçoit les variables JSON des clients Godot et les écrit dans un fichier texte réinitialisé à chaque session.

Communication : Broadcast des signaux de démarrage/arrêt à tous les clients connectés simultanément via TCP.

Pages de l'Interface

Menu d'accueil : Deux options - Gestion de session ou Statistiques.

Lancement de session : Configuration des paramètres (lieu obligatoire avec autocomplétion, complément optionnel, sélection du jeu, nombre de joueurs, temps de jeu). Validation par messageBox de confirmation.

Gestion de session : Affichage du jeu sélectionné, score avec unité correspondante, chronomètre qui décrémente avec changement de couleur dynamique, missions si le jeu les contient, messages prédéfinis, affichage des variables reçues. Boutons : Annuler, Démarrer/Terminer, Pause.

Fin de session : Récapitulatif complet (lieu, complément, jeu, joueurs, score final, temps écoulé). Bouton Sauvegarder qui insère les données en base de données.

Statistiques : Trois onglets - Liste de toutes les sessions avec pagination, graphiques (nombre de sessions par jeu, scores moyens, participants), résumé avec totaux.

À propos de ce projet

Ce projet de stage du 2 mars au 17 avril 2026 au sein de L'Approximatographe m'a permis de développer une vision globale d'un projet complet en m'obligeant à réfléchir au fonctionnement global et pas seulement aux parties isolées. J'ai progressé en autonomie, gestion de projet agile, architecture système distribuée et interaction client-serveur sur réseau local.
